# プロジェクト基本ルール / Project Base Rules

このファイルは、プロジェクト「たこさし ～令和最新版～」の開発方針やコーディング規約を定義するものです。AIアシスタント（Antigravity）や他の開発者が一貫性を保つために参照します。

## 1. プロジェクト概要
- **名称**: たこさし ～令和最新版～
- **ジャンル**: たこ焼き焼成アクションゲーム
- **コンセプト**: 令和の最新技術（Firebase, AI）を詰め込んだ、モダンで高品質なたこ焼きクリックゲーム。

## 2. デザイン・トーン
- **UI言語**: ユーザーに向けたテキストは全て**関西弁（大阪弁）**で記述する。
  - 例: 「ゲーム開始！」→「ゲーム開始！」(そのまま)、「名前を入力してや！」、「門前払い（もっと修行してや）」
- **ビジュアル**: プレミアムな外観を目指す。
  - グラデーション、ガラスモーフィズム、スムーズなアニメーションを活用。
  - 視認性が高く、かつリッチな配色にする。

## 3. 技術スタック
- **Frontend**: 単一の `index.html` (Vanilla HTML / CSS / JS)。
- **Styling**: `index.html` 内の `<style>` タグに記述。CSS変数を活用する。
- **Database**: Firebase Firestore (ハイスコア送信・取得)。
- **AI Integration**: Google Gemini API (Flashモデル) を使用したプレイヤー名の自動不適切判定。

## 4. コーディング規約
- **可読性**: JSのコードはできる限り整理し、役割ごとにコメントを入れる。
- **パフォーマンス**: 大量に生成されるたこ焼き要素の DOM 操作は効率的に行い、メモリリークを避ける。
- **レスポンシブ**: モバイルファーストで設計し、`touch-action: manipulation` でタップ感度を最適化する。
- **セキュリティ**: APIキーの扱いには注意する（現在の実装は学習・プロトタイプ用として許可されているが、本番時は制限をかける）。

## 5. ゲームロジックの要点
- **グリッド**: 5列の子要素（合計20個の穴）。
- **状態遷移**: 出現(step1) → 加熱(step2) → 完成(done) → 危険(warning) → 焦げ(burnt)。
- **スコア**:
  - 通常たこ焼き: 20点
  - 黄金(10%): 200点
  - 毒(15%): -100点
- **コンボ**: 5コンボごとにボーナス加算。焦げ・毒でリセット。
